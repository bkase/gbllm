# Generated Artifacts

Artifacts under `docs/review/f-b1/artifacts/` are generated by:

```bash
cargo run -p gbf-test --bin f_b1_regen
```

The review scripts call this after the fast and heavy F-B1 gates.

The representative `.gb` artifact is the streaming matmul ROM used for the
headline N=128 run. The full N sweep in `realism_report.v1.json` is generated
by running one loaded ROM per matrix size, tracing copy/compute yield safe
points plus the ROM VBlank handler for L4 frame-service evidence, stopping at
the repeated tile-safe PC, decoding the existing F-A3 `DumpArena` command shape
for the accumulator-tile read, and reassembling tiles on the host. `gbf-test`
owns the packet regeneration binary because it may depend on `gbf-verify` for
independent byte-exact conformance checks; production crates still do not
depend on `gbf-verify`.

`frame_trace_n128.summary.md` is the checked-in replacement for the raw
N=128 frame trace. The raw pretty JSON trace is generated transiently during
packet regeneration and consumed by the report checks, but it is not committed
because it is about 45 MiB. The summary preserves the review-relevant
invariants: VBlank provenance, widget/scheduler service counts, liveness
margin, and no-yield/no-pause-under-lease counts.

`run_latest.log` is the structured packet log for the latest regeneration, and
`run_latest.log.summary.json` is the diff-friendly count summary consumed by
reviewers and CI.
